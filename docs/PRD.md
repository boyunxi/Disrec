## 2. 背景与目标

### 2.1 背景

推荐算法太"懂"用户了——它只推用户喜欢看的、认同的、舒服的内容。久而久之，用户的世界观被算法越箍越窄：误以为全世界都和自己想的一样，温和的中间立场被极端声音淹没，不同群体之间的理解鸿沟越拉越大。

信息茧房不是用户主动选择的，而是算法为了最大化用户停留时长而系统性制造的。RSS、收藏夹、书单等工具虽然可以帮助用户主动获取多元信息，但操作成本太高，普通人难以坚持。用户需要一个"轻量级、低成本、可持续"的方式，来对抗算法带来的认知收窄。

反推荐引擎（Disrec — Disrecommendation Engine）正是为此而生：它和主流推荐系统反着来——不是让用户爽，而是让用户"醒"。每天主动给用户推送 3 条"用户平时绝对不会点，但可能对用户有用"的内容，帮助用户看到算法之外的世界。

### 2.2 目标

| 目标层级 | 描述 |
|---------|------|
| **核心目标** | 每天为用户推送 3 条经过多维度平衡筛选的异质内容，帮助用户打破信息茧房 |
| **体验目标** | 用户每天仅需 5 分钟即可完成当日"信息食谱"的摄入，操作成本极低 |
| **认知目标** | 通过持续的多元视角暴露，提升用户的批判性思维能力和认知多样性 |
| **产品形态目标** | 首期实现网页端工具，后续扩展为浏览器插件和公众号机器人 |

### 2.3 成功指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| 日活用户（DAU） | 每日至少阅读 1 条推荐内容的用户数 | 上线 1 个月达到 500 |
| 每日完成率 | 每日 3 条推荐全部阅读的用户占比 | ≥ 40% |
| 有用反馈率 | 标记"有用"的推荐内容占已阅读内容的比例 | ≥ 30% |
| 次日留存率 | 新用户次日再次访问的比例 | ≥ 50% |
| 认知多样性评分 | 用户一周内阅读内容在立场、证据、领域三个维度的综合评分 | 周均值 ≥ 65 分 |

## 3. 用户场景

### 3.1 目标用户

| 用户类型 | 典型特征 | 核心需求 |
|---------|---------|---------|
| **大学生、研究生、科研人员** | 做研究时容易陷入单一视角，文献检索容易形成"回音壁" | 快速获取反面论据、不同研究范式和跨学科视角 |
| **职场人士、产品经理、运营** | 需要理解不同圈层用户的真实想法，但自身信息源单一 | 低成本了解异质群体观点，补充用户同理心 |
| **普通网民** | 日常被算法推荐包围，意识到自己信息茧房但无力打破 | 用最轻量的方式接触"不舒服但有营养"的信息 |

### 3.2 用户故事

1. **作为研究生**，我希望每天看到一条与我研究方向立场相反的学术观点，这样我写论文时可以提前预判反驳意见。
2. **作为产品经理**，我希望反推荐引擎给我推送我不了解的用户群体的真实想法，这样我设计功能时不会只站在自己角度。
3. **作为普通网民**，我希望每天花 5 分钟看到一条我平时绝对不会点的内容，并附上"为什么算法不会推给我"的解释，这样我能意识到自己被什么信息环境包裹着。
4. **作为通勤用户**，我希望在地铁上快速浏览完今天的 3 条推荐并标记反馈，这样我的认知多样性可以像每天吃维生素一样轻松维持。
5. **作为长期用户**，我希望看到一周的认知多样性报告，这样我能直观感受到自己的信息食谱是否均衡。

### 3.3 使用场景

| 场景 | 时间 | 行为 | 期望体验 |
|------|------|------|---------|
| **通勤阅读** | 早晨通勤 5-10 分钟 | 打开反推荐引擎，快速浏览今日 3 条推荐，标记有用/不感兴趣 | 加载快、布局清晰、单手可操作 |
| **研究辅助** | 写论文/做方案前 10-15 分钟 | 生成今日食谱，重点阅读与自己立场相反的内容，展开阅读详细论述 | 推荐理由清晰，立场和证据维度明确标注 |
| **睡前替代** | 睡前 5-10 分钟 | 替代短视频，用反推荐引擎接触一条"不舒服但有营养"的信息 | 深色主题护眼，内容不过度刺激情绪 |
| **周度回顾** | 周末 5 分钟 | 查看本周认知多样性报告，了解自己的阅读分布 | 数据可视化直观，给出具体改进建议 |

### 4.2 功能详情

#### F01 每日 3 条推荐内容卡片

**描述：** 每日生成 3 条经过多维度平衡筛选的推荐内容，以卡片形式展示。3 条内容分别通过算法平衡、编辑精选、随机发现三种机制选出，确保推荐结果的多样性和不可预测性。

**交互流程：**
1. 用户进入首页，自动触发推荐生成
2. 系统依次选出编辑精选（高证据质量）、随机发现（引入随机性）、算法平衡（维度互补）各 1 条
3. 3 条内容随机排序后以卡片列表展示
4. 用户可点击"生成今日食谱"按钮手动刷新推荐
5. 用户可通过筛选按钮（全部/算法平衡/编辑精选/随机发现）过滤卡片

**页面元素：**
- 卡片区域标题："今日信息食谱"，副标题显示"3 条经过多维度平衡筛选的内容"
- 筛选按钮组：全部、算法平衡、编辑精选、随机发现
- 每张卡片包含：选择机制标签、话题标签、来源、序号、标题、摘要、推荐理由、展开阅读区域（详细内容+六维度标签+话题标签）、展开/收起按钮、反馈按钮

**异常处理：**
- 内容库不足 3 条时：补充高证据质量内容，确保始终有 3 条推荐
- 筛选条件下无匹配内容时：显示"当前筛选条件下暂无内容"的空状态提示
- 同一天多次刷新：每次刷新重新随机生成，不保留历史推荐

#### F02 多维度内容标签系统

**描述：** 每条推荐内容在立场、证据、情绪、时间尺度、地理范围、来源类型六个维度上进行标注，用户展开阅读时可查看完整的维度信息。

**交互流程：**
1. 卡片默认折叠状态，仅显示标题和摘要
2. 用户点击"展开阅读"后，卡片展开显示详细内容和六维度标签网格
3. 立场维度以"保守/反对 ~ 激进/支持"文字 + 数值展示
4. 证据等级以星级（1-5 星）展示
5. 情绪基调以文字展示（正面/希望、中性/客观、负面/忧虑）
6. 时间尺度、地理范围、来源类型以文字标签展示

**页面元素：**
- 六维度网格（2×3 布局，移动端 2 列，PC 端 3 列）
- 每个维度格子：维度名称（灰色小字）+ 维度值（白色/彩色文字）
- 证据等级：★☆ 图标，4 星以上绿色、3 星黄色、2 星以下橙色

**维度定义：**

| 维度 | 取值范围 | 说明 |
|------|---------|------|
| stance（立场） | -2 ~ +2 | -2=保守/悲观/反对，0=中性/平衡，+2=激进/乐观/支持 |
| evidence（证据） | 1 ~ 5 | 5=最高证据强度（学术研究/机构报告），1=最低（个人叙事） |
| emotion（情绪） | -2 ~ +2 | -2=负面/焦虑，0=中性，+2=正面/希望 |
| timeScale（时间尺度） | 短期/中期/长期 | 内容关注的时间维度 |
| geography（地理范围） | 本地/区域/全球 | 内容关注的地理维度 |
| sourceType（来源类型） | 主流媒体/独立媒体/学术研究/机构报告/个人叙事 | 内容来源分类 |

#### F03 信息食谱均衡度面板

**描述：** 在首页 Hero 区域右侧展示当日 3 条推荐内容在立场多样性、证据质量、领域跨度三个维度的综合评分，帮助用户直观了解今日"信息食谱"的均衡程度。

**交互流程：**
1. 页面加载/刷新推荐时，自动计算并更新均衡度评分
2. 三个维度各以进度条展示，实时动态填充
3. 顶部显示综合均衡度评分（满分 100）

**页面元素：**
- 综合评分：大号数字（青色），右侧显示"今日信息食谱均衡度"标签
- 立场多样性：标签 + 进度条（青-蓝渐变）
- 证据质量：标签 + 进度条（紫渐变）
- 领域跨度：标签 + 进度条（粉-玫红渐变）
- 用户兴趣基线标签组：显示用户已选的兴趣标签

**评分算法：**
- 立场多样性：基于 3 条内容立场值的极差计算，范围越大分数越高
- 证据质量：基于 3 条内容证据等级的均值计算
- 领域跨度：基于话题和来源类型的去重数量计算
- 综合评分 = 三者均值

#### F04 推荐透明度面板

**描述：** 展示当日推荐结果的组成信息，让用户了解推荐是怎么生成的、覆盖了哪些领域和来源。

**交互流程：**
1. 页面加载/刷新推荐时，自动更新透明度面板
2. 面板以列表形式展示四项信息

**页面元素：**
- 推荐组成：编辑 N · 算法 N · 随机 N
- 覆盖领域：逗号分隔的话题列表
- 来源类型：逗号分隔的来源类型列表
- 平均证据等级：X.X / 5（青色高亮）

#### F05 观点光谱可视化

**描述：** 将当日 3 条推荐内容在保守-激进光谱上的位置以圆点形式展示，帮助用户直观感知推荐内容的立场分布。

**交互流程：**
1. 页面加载/刷新推荐时，自动更新光谱上的圆点位置
2. 鼠标悬停在圆点上，显示内容标题和立场数值

**页面元素：**
- 光谱底图：一条水平线，左侧标注"保守"，中间标注"温和"，右侧标注"激进"
- 3 个圆点：分别以青色、紫色、粉色标识
- 悬停提示：标题 + 立场数值

**映射规则：** stance -2 映射到最左，+2 映射到最右，0 映射到中间。

#### F06 兴趣基线设置

**描述：** 用户可设置自己平时关注的话题标签作为"兴趣基线"，算法以此为参考寻找多元视角，而不是把用户困在这些话题里。

**交互流程：**
1. 用户点击 Header 区域的设置图标
2. 弹出设置弹窗，展示可选标签列表
3. 已选标签高亮显示，未选标签灰暗显示
4. 用户点击标签切换选中/取消选中状态
5. 至少保留 1 个标签，不允许全部取消
6. 用户点击"保存设置"按钮，关闭弹窗并刷新推荐

**页面元素：**
- 弹窗标题："设置你的兴趣基线"
- 说明文案："选择你平时关注的话题，算法会以此为参考寻找多元视角，而不是把你困在这些话题里。"
- 标签按钮组：8 个可选标签（科技数码/效率工具/AI 资讯/创业思维/财经投资/人文历史/艺术设计/健康生活）
- 保存按钮："保存设置"（渐变背景，全宽）

**预设标签：**

| 标签名 | 颜色标识 | 默认选中 |
|--------|---------|---------|
| 科技数码 | cyan | ✓ |
| 效率工具 | violet | ✓ |
| AI 资讯 | purple | ✓ |
| 创业思维 | blue | ✓ |
| 财经投资 | emerald | ✗ |

#### F07 推荐理由解释

**描述：** 每条推荐内容卡片中附带"为什么推荐这条？"的理由说明，帮助用户理解推荐的逻辑。

**交互流程：**
1. 卡片默认折叠状态下即可看到推荐理由区域（卡片内的彩色左边框区块）
2. 理由根据内容的选择机制、话题独特性、来源多样性、证据质量动态生成

**页面元素：**
- 彩色左边框区块（颜色与选择机制标签一致）
- 小标题："为什么推荐这条？"（彩色文字）
- 理由正文（白色文字）

**理由生成逻辑：**
- 编辑精选 → "由编辑人工筛选，证据质量高、视角独特"
- 随机发现 → "通过随机机制引入，防止推荐结果被算法惯性锁死"
- 算法平衡 → "由算法基于多维度平衡选出"
- 话题唯一 → 追加"补充了「{话题}」这一未被其他推荐覆盖的领域"
- 来源唯一 → 追加"引入了{来源类型}视角，丰富来源多样性"
- 证据高（≥4）→ 追加"证据等级较高，适合建立可靠认知"

#### F08 内容反馈

**描述：** 用户可以对每条推荐内容进行"有用"或"不感兴趣"的反馈，反馈将影响后续推荐策略。

**交互流程：**
1. 卡片底部右侧展示两个反馈按钮
2. 用户点击"有用"按钮：按钮变为绿色，文字变为"已标记有用"
3. 用户点击"不感兴趣"按钮：按钮变暗，文字变为"已标记不感兴趣"
4. 反馈为互斥操作，点击一个后另一个不可点击（当前版本）

**页面元素：**
- "有用"按钮：圆角胶囊形，默认灰色背景
- "不感兴趣"按钮：圆角胶囊形，默认灰色背景
- 点击后状态变化：文字更新 + 颜色变化

#### F09 认知盲区分析模块（新增）

**描述：** 基于用户的兴趣基线设置和阅读历史（反馈数据），分析用户可能存在的认知盲区，并在首页醒目位置展示分析结果。

**交互流程：**
1. 系统基于用户已选兴趣标签，推断用户高频接触的话题领域
2. 对比内容库的全量话题分布，识别用户兴趣基线未覆盖的领域
3. 基于用户"不感兴趣"反馈的分布，识别用户回避的立场/情绪维度
4. 在首页 Hero 区域下方新增"认知盲区"卡片，展示分析结果
5. 点击某个盲区标签，自动筛选相关内容供用户阅读

**页面元素：**
- 独立卡片，位于信息食谱均衡度面板下方
- 卡片标题："你的认知盲区"，副标题："基于你的兴趣基线和阅读历史分析"
- 盲区标签列表：以红色/橙色警告色展示未覆盖的话题领域
- 每个标签附带简要说明，如"你很少接触「老龄化和数字鸿沟」相关话题"
- 回避维度提示：如"你倾向于回避负面情绪的内容"或"你的阅读偏向保守立场"
- 每个盲区标签可点击，跳转并筛选相关内容

**分析维度：**
- 话题盲区：兴趣基线未覆盖的内容话题
- 立场盲区：用户回避的立场方向
- 情绪盲区：用户回避的情绪类型
- 来源盲区：用户很少接触的来源类型

**异常处理：**
- 首次使用无阅读历史时：仅基于兴趣基线分析话题盲区
- 兴趣基线已覆盖所有话题时：提示"你的兴趣基线很全面！但仍可能有立场和来源的盲区"

#### F10 推荐理由增强（新增）

**描述：** 每条推荐内容附带的推荐理由从单一解释增强为两个维度："为什么算法不会推给你？"和"你的认知盲区可能在哪里？"。

**交互流程：**
1. 在原有推荐理由区块下方，新增两个解释区块
2. "为什么算法不会推给你？"：解释主流推荐算法为何不会推送此类内容
3. "你的认知盲区可能在哪里？"：指出用户在该话题上可能的认知局限
4. 两个区块以不同的视觉样式区分

**页面元素：**
- 区块一（橙色调左边框）：
  - 小标题："为什么算法不会推给你？"
  - 正文：如"主流推荐算法倾向于推送与你兴趣标签匹配的内容，而该内容属于你未关注的「{话题}」领域，不会出现在你的信息流中。"
- 区块二（紫色调左边框）：
  - 小标题："你的认知盲区可能在哪里？"
  - 正文：如"你可能从未从{立场方向}的角度思考过{话题}，这可能导致你对该议题的理解偏于单一。"

**生成逻辑：**

| 场景 | "为什么算法不会推给你？" | "你的认知盲区可能在哪里？" |
|------|------------------------|--------------------------|
| 话题不在兴趣基线 | "该内容属于你未关注的「{话题}」领域" | "你对{话题}的了解可能仅来自二手转述" |
| 立场与用户历史偏好相反 | "算法会过滤掉与你过往阅读立场相反的内容" | "你可能从未认真考虑过{立场方向}的论据" |
| 来源类型用户未接触过 | "算法优先推送你常读的{来源类型}来源" | "你可能缺少{来源类型}提供的一手视角" |
| 情绪维度用户回避 | "算法不会推送你倾向回避的{情绪}内容" | "你可能在{情绪}信息面前缺乏"心理免疫力"" |

**异常处理：**
- 内容与用户兴趣高度重合时："这条内容与你兴趣相关，但算法可能因为其{维度}特征而降低排序权重"
- 新用户无历史数据时：仅基于兴趣基线生成理由

#### F11 历史记录与数据持久化（新增）

**描述：** 使用浏览器 localStorage 保存用户的兴趣基线设置、阅读记录和反馈数据，实现跨会话数据持久化。

**交互流程：**
1. 用户修改兴趣基线设置后，自动保存到 localStorage
2. 用户展开阅读某条内容，自动记录该内容的阅读时间到 localStorage
3. 用户提交反馈（有用/不感兴趣），自动记录反馈数据到 localStorage
4. 用户重新访问页面时，自动从 localStorage 恢复设置和历史数据
5. 每日推荐内容按日期存储，同一天内刷新页面不重复生成

**数据结构：**

```
localStorage keys:
- disrec_user_tags        // 用户兴趣基线标签数组 JSON
- disrec_theme            // 主题偏好：'dark' | 'light'
- disrec_daily_{date}     // 当日推荐数据 JSON（含阅读状态和反馈）
- disrec_history          // 阅读历史摘要 JSON（最近 30 天）
```

**存储上限策略：**
- 阅读历史仅保留最近 30 天数据，超期自动清理
- 单日推荐数据结构不超过 10KB
- 总存储量控制在 500KB 以内

**异常处理：**
- localStorage 不可用（隐私模式/存储已满）：降级为内存存储，页面关闭后数据丢失，并在页面底部提示"当前浏览器不支持数据持久化，你的设置不会保存"
- 数据格式异常（版本升级导致结构变化）：清除旧数据，使用默认值初始化

#### F12 每日进度追踪（新增）

**描述：** 追踪用户今日 3 条推荐内容的阅读状态（未读/已读），并在页面顶部展示完成进度。

**交互流程：**
1. 每日 3 条推荐生成后，初始状态均为"未读"
2. 用户展开阅读某条内容后，该条状态自动变为"已读"，记录阅读时间
3. 顶部进度指示器实时更新：0/3 → 1/3 → 2/3 → 3/3
4. 全部阅读完成后，进度指示器展示庆祝动画和鼓励文案
5. 进度状态通过 localStorage 持久化，刷新页面不丢失

**页面元素：**
- 进度指示器：位于 Header 下方或卡片列表上方
- 环形进度条或线性进度条 + 文字："今日进度 1/3"
- 3/3 完成状态：进度条满格 + "今日认知食谱已摄入完毕 🎯"文案
- 每张卡片的未读/已读标识：未读卡片左上角显示小圆点

**异常处理：**
- 用户手动刷新推荐：重置进度为 0/3
- 跨日自动重置：新的一天自动清除昨日进度

#### F13 认知多样性周报（新增）

**描述：** 汇总用户一周内的阅读数据，生成认知多样性报告，帮助用户了解自己的信息摄入模式和认知均衡程度。

**交互流程：**
1. 用户点击底部"本周报告"入口或完成 7 天阅读后系统自动提示
2. 展示周报页面/弹窗，包含以下数据：
   - 本周阅读量（总条数）
   - 立场分布图（饼图/柱状图，展示各立场占比）
   - 话题分布图（展示覆盖的话题领域）
   - 认知多样性综合评分（与上周对比）
   - 本周认知盲区提示
   - 改进建议
3. 用户可关闭周报，返回主页面

**页面元素：**
- 周报入口按钮：底部或侧边栏
- 周报弹窗/页面：
  - 标题："认知多样性周报"
  - 时间范围：{周一日期} - {周日日期}
  - 核心指标：阅读量、多样性评分（环形图）
  - 立场分布图：展示各立场占比的柱状图
  - 话题分布图：展示各话题覆盖情况的热力图/标签云
  - 盲区提示：标注用户本周未触及的维度
  - 改进建议：如"下周试试多读一些来自学术研究来源的内容"

**数据来源：**
- 基于 localStorage 中最近 7 天的阅读历史和反馈数据计算
- 无足够数据时（阅读天数 < 3 天）显示"数据不足，继续使用一周后即可查看完整报告"

**异常处理：**
- 新用户第一周无完整数据：显示"这是你的第一周，数据积累中…"
- 数据缺失（用户清除了 localStorage）：按实际可用数据生成报告，标注数据不完整

#### F14 分享功能（新增）

**描述：** 支持用户将推荐内容卡片分享到社交媒体，扩大产品影响力。

**交互流程：**
1. 每张卡片底部新增"分享"按钮

**页面元素：**

**异常处理：**
- 浏览器不支持 Web Share API：降级为"复制链接"方式

#### F15 主题切换（新增）

**交互流程：**

#### F16 响应式移动端优化（新增）

**描述：** 对现有页面进行响应式适配，确保在移动端（320px ~ 768px）和桌面端（768px+）均有良好体验。

**交互流程：**
1. 用户在移动端浏览器访问页面
2. 页面自动适配为单列布局
3. 所有交互元素尺寸适配触摸操作（最小点击区域 44×44px）
4. 卡片展开/收起操作对触摸友好

**适配要点：**

| 组件 | 桌面端 | 移动端 |
|------|--------|--------|
| Hero 区域 | 双列布局（左文右面板） | 单列堆叠（文上面板下） |
| 透明度+光谱 | 双列并排 | 单列堆叠 |
| 卡片维度网格 | 3 列 | 2 列 |
| 筛选按钮组 | 横排 | 横排可滚动或换行 |
| 设置弹窗 | 居中弹窗 | 底部抽屉或全屏弹窗 |
| 字号 | 标题 3xl / 正文 sm | 标题 2xl / 正文 sm |
| 间距 | 标准间距 | 适当收紧 |

**异常处理：**
- 极窄屏幕（< 320px）：确保内容不溢出，使用 min-width 兜底
- 横屏模式：重新计算布局断点

## 8. 风险评估

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|---------|
| 内容库规模有限，推荐重复率高 | 高 | 中 | 扩充内容库至 50+ 条；引入去重逻辑，避免连续两天推荐相同内容 |
| 用户对异质内容抵触，流失率高 | 高 | 高 | 推荐理由充分解释"为什么"；初期控制异质程度，渐进式增加；"有用/不感兴趣"反馈帮助调节 |
| 纯前端无用户账号体系，无法跨设备同步 | 中 | 中 | 当前阶段接受该限制，后续版本引入轻量账号体系或浏览器同步 |
| localStorage 容量限制导致数据丢失 | 低 | 高 | 实施 30 天数据自动清理策略；关键数据压缩存储 |
| 浅色主题适配工作量大 | 中 | 低 | 使用 CSS 变量统一管理主题色，降低适配成本 |
| 推荐算法纯前端实现，无法利用服务端算力 | 中 | 中 | 当前内容规模下前端计算完全足够；规模扩大后迁移推荐逻辑到后端 |
| 内容立场标注主观性 | 中 | 中 | 多人交叉标注；对争议标注降权处理；在透明度面板公开标注方法 |

### 10.1 术语表

| 术语 | 定义 |
|------|------|
| Disrec | Disrecommendation Engine 的缩写，即反推荐引擎 |
| 信息食谱 | 借用营养学概念，将信息摄入比喻为饮食，强调均衡和多样性 |
| 信息茧房 | 用户因算法推荐而长期接触同质信息，导致认知范围收窄的现象 |
| 认知盲区 | 用户因信息来源单一而未能意识到的知识空白或视角缺失 |
| 兴趣基线 | 用户设置的日常关注话题标签，作为反推荐算法的输入参考 |
| 算法平衡 | 通过多维度评分选出与已有内容形成维度互补的推荐机制 |
| 编辑精选 | 由人工筛选的高证据质量、独特视角内容的推荐机制 |
| 随机发现 | 引入随机性以防止算法自我强化的推荐机制 |
| 立场维度 | 内容在保守/反对 ~ 激进/支持光谱上的位置，取值 -2 ~ +2 |
| 证据等级 | 内容的论据可靠性强度，取值 1 ~ 5，5 为最高 |
| 认知多样性 | 用户信息摄入在立场、证据、领域等多个维度上的均衡程度 |
